1. Introduction

1.1 Purpose

This web application is meant to help its users keep track of household/organization’s resources and notify them through emails and in-app notices about the need to either replenish stocks of specific items, maintenance check-ups, and buy/replace spare parts.

1.2 Intended Audience and Reading Suggestions

The intended audience for the web application are household and organizational managers

1.3 Product Scope

The web application provides information and statistics about specific resources and technical deadlines/necessities of a user. The main objective of the app is to provide a easy-to-use record-holder of their household needs which provides timely reminders to keep their stocks full and devices working in the best way.

2. Overall Description

2.1 Product Functions

The app will be able to keep track of the items and dates that have been imported by the user and provide reminders in the form of in-app notices and emails to update the database, refill the stocks of specific items/categories, and/or provide maintenance or check up on different devices.

2.2 User Classes and Characteristics

Unauthorized User (guest)

users that don’t have an account and/or are first-time visitors of the app
  • can view the landing page and the overall website without in-use databases
  • can access the login and register pages

Authorized User

users that have logged into their accounts
  • can access the home page
  • can access the databases they have imported
  • can import and export databases
  • can generate/download statistic
  • can set time-limits and reminders
  • can access the personal cabinet where they can change their personal data

2.3 Operating Environment

The operating environment for the app will be any browser with support for HTML5 (FireFox, Chrome, Safari, etc.) on any operating system. (Mindows, Mac OS, etc.)

2.4 Design and Implementation Constraints

Developers have encountered problems with sending notice emails through Gmail because of their update security policies.

2.5 User Documentation

The users of the application will have informations about the features of the app and how to use it properly on the landing page. Such information are: how to insert/delete/update their supplies manually into database, how to import/export their data using defined format, how to change their password and downloading statistics.

2.6 Assumptions and Dependencies

To make possible the import/export feature, an additional library will be selected from the available open source ones.

3. External Interface Requirements

3.1 User Interfaces

The application interface provides easy-to-use buttons (save, edit, delete, update). It also provides notifications regarding the resupply date of each item and error messages when inserting wrong type of data (inserting wrong email, password, data fields and attributes);

3.2 Communication Interfaces

In order to ensure comunication, the web application will use Hypertext Transfer Protocol (HTTP)

4. System Features

4.1 Import Data

The user will be able to import data if they already have a CSV/XML/JSON format (having the following attributes -> name, quantity, unit, type, supply, notice ) of the items they want to keep track of and/or introduce the data manually.

4.2 Export Data

The user will be able to export the data they have introduced into the app’s database as CSV/XML/JSON file, having the same attributes.

4.3 Generate Statistics

The user will have the option to generate statistics based on number of noticies in each month and the number of supplies in the Resource category (Food, Fuel, Make-Up, Tools, Medicine, Office Supplies) or Maintenance category(Spare Parts, Insurance, Check Up) and view them as HTML or PDF files.

4.4 In-app Notices

The application will provide in-app pop-up notices of the list of items from each category that exceeded their notice deadline for resupply and/or necessary maintenance.
